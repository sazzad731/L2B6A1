# TypeScript ইন্টারভিউ প্রশ্নাবলী (বাংলা)



## ১. TypeScript-এ Interface এবং Type-এর মধ্যে পার্থক্য

TypeScript-এ Interface এবং Type দুটোই কাস্টম টাইপ তৈরি করতে ব্যবহৃত হয়, কিন্তু তাদের ব্যবহারের ধরন ও ক্ষমতায় কিছু পার্থক্য আছে। Interface সাধারণত অবজেক্টের গঠন বা কাঠামো নির্ধারণ করতে ব্যবহৃত হয়, তবে Type আরও বেশি বহুমুখী এবং অবজেক্ট ছাড়াও union, tuple, primitive ইত্যাদি বিভিন্ন রকম টাইপ তৈরি করতে পারে। Interface পুনরায় ডিক্লেয়ার করলে স্বয়ংক্রিয়ভাবে merge হয়ে যায়, কিন্তু Type পুনরায় ডিক্লেয়ার করলে error দেয়। Interface “extends” ব্যবহার করে একাধিক interface থেকে বৈশিষ্ট্য নিতে পারে, আর Type “intersection” ব্যবহার করে একাধিক টাইপ মিলিয়ে নতুন টাইপ তৈরি করতে পারে। সাধারণভাবে, অবজেক্ট স্ট্রাকচারের জন্য Interface বেশি ব্যবহার হয় এবং জটিল বা বিভিন্ন ধরনের টাইপ তৈরির ক্ষেত্রে Type বেশি সুবিধাজনক।


## ২. TypeScript-এ any, unknown এবং never টাইপের পার্থক্য

TypeScript-এ any, unknown, এবং never—এই তিনটি টাইপের উদ্দেশ্য ও ব্যবহার একে অপরের থেকে সম্পূর্ণ আলাদা। “any” হলো সবচেয়ে flexible টাইপ, যা টাইপ চেকিং বন্ধ করে দেয় এবং যেকোনো মান গ্রহণ করতে দেয়, তবে এটি ব্যবহারে টাইপ সেফটি নষ্ট হয়। “unknown” দেখতে any-এর মতো হলেও এটি নিরাপদ, কারণ কোনো অপারেশন করার আগে অবশ্যই টাইপ চেক করতে হয়। এটি any-এর একটি সেফ বিকল্প। অন্যদিকে “never” এমন টাইপকে বোঝায় যেখানে কোনো মান কখনোই থাকতে পারে না। এটি সাধারণত এমন ফাংশনে ব্যবহৃত হয় যা কখনো কোনো মান রিটার্ন করে না, অথবা এমন পরিস্থিতিতে যেখানে কোডের কোনো অংশে পৌঁছানো যুক্তিগতভাবে অসম্ভব। এই তিনটি টাইপের পার্থক্য বোঝা টাইপ সেফটি নিশ্চিত করতে গুরুত্বপূর্ণ।
