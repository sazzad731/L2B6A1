TypeScript ইন্টারভিউ প্রশ্নাবলী (বাংলা)

এই ডকুমেন্টটি TypeScript-এর কিছু গুরুত্বপূর্ণ ইন্টারভিউ প্রশ্নের উত্তর নিয়ে তৈরি করা হয়েছে, যা আপনার প্রযুক্তিগত জ্ঞানকে আরও শক্তিশালী করতে সাহায্য করবে।

১. TypeScript-এ interface এবং type-এর মধ্যে কিছু পার্থক্য কী কী?

interface এবং type উভয়ই TypeScript-এ কাস্টম ডেটা স্ট্রাকচার (custom data structures) সংজ্ঞায়িত করতে ব্যবহৃত হয়। তবে এদের কাজের ধরণ এবং ব্যবহারের ক্ষেত্রে কিছু মৌলিক পার্থক্য রয়েছে:

interface

type

ডিক্লিয়ারেশন মার্জ (Declaration Merging)

সম্ভব (Possible)। একই নামের একাধিক interface ডিক্লিয়ার করলে TypeScript সেগুলোকে স্বয়ংক্রিয়ভাবে একটি একক কাঠামোতে মার্জ করে দেয়। এটি লাইব্রেরি ডেভলপমেন্টে খুবই উপযোগী।

সম্ভব নয় (Not Possible)। একই নামে একাধিক type ডিক্লিয়ার করা যায় না।

এক্সটেনশন (Extension)

এটি extends কিওয়ার্ড ব্যবহার করে অন্য interface বা type থেকে বৈশিষ্ট্য উত্তরাধিকার (inherit) নিতে পারে।

এটি & (Intersection) অপারেটর ব্যবহার করে অন্য type বা interface-এর সাথে মিলিত হতে পারে।

ব্যবহারের পরিসর

প্রধানত অবজেক্টের কাঠামো (Object Shapes), ক্লাস ইমপ্লিমেন্টেশন এবং ফাংশনের স্বাক্ষর (function signatures) সংজ্ঞায়িত করার জন্য ব্যবহৃত হয়।

একটি ক্লাস implements কিওয়ার্ড ব্যবহার করে একটি interface-কে ইমপ্লিমেন্ট করতে পারে।

একটি ক্লাস সরাসরি একটি type-কে ইমপ্লিমেন্ট করতে পারে না, যদি না সেই type একটি ইন্টারফেসের অনুরূপ অবজেক্ট কাঠামো সংজ্ঞায়িত করে।

উদাহরণ: ডিক্লিয়ারেশন মার্জিং (Interface)

// প্রথম ডিক্লিয়ারেশন
interface Person {
  name: string;
}

// দ্বিতীয় ডিক্লিয়ারেশন, এটি প্রথমটির সাথে মার্জ হবে
interface Person {
  age: number;
}

// এখন Person ইন্টারফেসে name এবং age উভয়ই রয়েছে
const user: Person = { name: "Rumi", age: 30 }; 


২. TypeScript-এ keyof কিওয়ার্ডের ব্যবহার কী? একটি উদাহরণ দিন।

keyof এর ব্যবহার

TypeScript-এ keyof কিওয়ার্ডটি একটি নির্দিষ্ট অবজেক্ট টাইপ বা ইন্টারফেসের সমস্ত প্রপার্টি নাম (property names) বা কী (Keys) নিয়ে একটি ইউনিয়ন টাইপ (Union Type) তৈরি করতে ব্যবহৃত হয়।

উদ্দেশ্য: এটি মেটা-প্রোগ্রামিং (meta-programming) এবং বিশেষ করে টাইপ সেফটি (Type Safety) নিশ্চিত করতে ব্যবহৃত হয়। keyof ব্যবহার করে আপনি কম্পাইল টাইমে (compile time) নিশ্চিত করতে পারেন যে আপনি প্যারামিটার হিসাবে যে স্ট্রিংটি দিচ্ছেন, সেটি অবজেক্টটির একটি বৈধ কী।

উদাহরণ:

আমরা একটি জেনেরিক ফাংশন তৈরি করব যা একটি অবজেক্ট এবং সেই অবজেক্টের একটি কী (Key) গ্রহণ করে এবং সেই কী এর মান (Value) প্রদান করে।

// একটি ইন্টারফেস সংজ্ঞায়িত করা হলো
interface Product {
  id: number;
  title: string;
  price: number;
}

// keyof Product হবে Union Type: 'id' | 'title' | 'price'

// জেনেরিক ফাংশন যা টাইপ সেফটি সহ যেকোনো প্রপার্টির মান খুঁজে নেয়
function getPropertyValue<T, K extends keyof T>(obj: T, key: K): T[K] {
  // K extends keyof T নিশ্চিত করে যে 'key' আর্গুমেন্টটি T অবজেক্টের একটি বৈধ প্রপার্টি নাম।
  // T[K] হচ্ছে লুকআপ টাইপ, যা সেই কী-এর সাথে সম্পর্কিত ভ্যালুর সঠিক টাইপ রিটার্ন করে।
  return obj[key];
}

const laptop: Product = {
  id: 404,
  title: "XPS Laptop",
  price: 1500,
};

// বৈধ ব্যবহার - টাইপ সেফটি নিশ্চিত
const laptopTitle = getPropertyValue(laptop, 'title'); // laptopTitle এর টাইপ হবে string
console.log(laptopTitle); // XPS Laptop

// অবৈধ ব্যবহার - কম্পাইল টাইমে ত্রুটি দেবে, কারণ 'color' Product এর কী নয়
// const laptopColor = getPropertyValue(laptop, 'color'); 


এইভাবে, keyof ব্যবহার করে ডেটা অ্যাক্সেস করার সময় আমরা শক্তিশালী টাইপ চেকিং নিশ্চিত করতে পারি।
